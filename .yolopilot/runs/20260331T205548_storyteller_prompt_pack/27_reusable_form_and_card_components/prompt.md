# 27 — Reusable Form and Card Components

Read `base_prompt.md` first and preserve all prior prompt work.

## Objective

Build the reusable UI pieces that will support tone selection cards, pitch cards, character sheets, and beat-sheet editors.

## Build
- Create card components, selection states, empty-state blocks, inline help text, and summary panels.
- Build consistent form components for selects, sliders, toggles, numeric fields, text areas, and validation messages.
- Add a small library of layout helpers for two-column forms, sticky side summaries, and split cards where useful.

## Deliverables

- Reusable card/form component library
- Documentation or story examples if you use a component explorer
- Examples of selected versus unselected cards

## Acceptance checks

- Future prompts can compose the workflow from reusable parts rather than one-off markup.
- Selection cards feel appropriate for genres, tones, pitches, and character sheets.
- Validation and helper text are visually consistent.

## Notes

Build just enough system to accelerate the workflow screens.
