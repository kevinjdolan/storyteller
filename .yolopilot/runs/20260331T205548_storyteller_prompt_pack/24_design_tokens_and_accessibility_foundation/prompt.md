# 24 — Design Tokens and Accessibility Foundation

Read `base_prompt.md` first and preserve all prior prompt work.

## Objective

Establish visual and accessibility primitives early so the later workflow screens remain consistent and usable.

## Build
- Define basic spacing, typography, color, surface, and radius tokens or theme values.
- Create reusable accessible building blocks for buttons, text inputs, panels, badges, progress bars, and lists.
- Set keyboard focus styles, semantic heading structure, and reduced-motion awareness where it matters.

## Deliverables

- Theme/token setup
- Reusable primitive components
- Accessibility checklist note

## Acceptance checks

- The project has a visual language before complex screens arrive.
- Interactive controls are keyboard visible and reasonably accessible.
- The design choices fit a calm bedtime-story product rather than a generic dashboard.

## Notes

Favor calm, readable UI over novelty.
