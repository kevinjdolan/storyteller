# 98 — Demo Data and First-Run Onboarding

Read `base_prompt.md` first and preserve all prior prompt work.

## Objective

Make the app easier to explore by providing seeded demo sessions or an onboarding hint flow for new developers and first-time users.

## Build
- Create optional demo data that shows at least one in-progress session and one completed session with representative metadata.
- Add a light onboarding empty state on the home screen that explains the workflow in plain language.
- Document how to load or reset the demo data during development.

## Deliverables

- Demo data seed or fixture flow
- Improved first-run empty state
- Docs for loading demo content

## Acceptance checks

- A fresh developer can see what the product is supposed to feel like quickly.
- The home screen is informative even when no sessions exist.
- Demo data does not become required for normal operation.

## Notes

This is about lowering friction.
